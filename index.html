Confirmed vs machine. Since Apple Mail Privacy Protection (2021) pre-loads every image on delivery, a raw "open" is unreliable — so each hit is classified:
• Confirmed — looks like a real person (e.g. opened in Gmail/webmail, or well after you sent it).
• Machine / prefetch — Apple MPP, a corporate security scanner, or anything that fired within ~2 min of your send time. These are shown but kept out of the confirmed count.
• Clicks are the strongest signal — Apple and most scanners don't follow links, so a click is almost always a real human. Prefer tracked links over opens when you can.
• Setting a send time sharpens the timing check; image blocking still means some real opens are never recorded at all. Treat everything as a strong hint, not proof.
Please use responsibly. Tracking opens/clicks may require disclosure or consent in some places (e.g. the EU/UK). Track your own correspondence, not people who'd object.

How do I deploy the backend? (~5 minutes, free)

Create the KV storage first — you can only bind it to the Worker once it exists.

  1. Create a free Cloudflare account (no card needed).
  2. Storage & Databases → KV → Create a namespace called pixel_data.
  3. Workers & Pages → Create → Worker → Start with Hello World. Name it pixel-track, Deploy. (Don't use "Upload files" — that's the static-site uploader.)
  4. Edit code, select-all & delete the sample, paste in worker.js, Deploy.
  5. Settings → Bindings → Add → KV namespace. Two fields: Variable name = PIXELS (exactly), KV namespace = pixel_data.
  6. Settings → Variables and Secrets → Add DASH_TOKEN = a long random password.
  7. Copy your Worker URL (https://pixel-track.you.workers.dev); test it at /health. Paste the URL + token above.

Welcome to Pixel 👋

An open-source, privacy-focused email tracking-pixel generator. Here's the gist.

How it works

  1. Create a pixel (and optional tracked links) here.
  2. Paste the snippet into your email as HTML, and send.
  3. Come back to see opens & clicks — with Apple/scanner prefetches filtered out so the numbers mean something.

You'll need a free Cloudflare Worker

A static site can show a pixel but can't record who loaded it — so tracking runs on a tiny Cloudflare Worker you deploy once (free, no credit card, ~5 min). Nothing runs on our servers.

One quick permission

For the best experience, allow Pixel to look up your own public IP via ipify (a free service). It's used only in your browser to hide hits that came from you (your own testing / opening your own mail), so the dashboard shows real recipient activity. Your IP is never sent to us — we don't collect anything. You can decline, but we don't recommend it (your own opens will clutter the results).
